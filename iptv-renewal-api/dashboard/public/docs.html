IPTV Renewal API

API RESTful para renovação automática de clientes em painéis IPTV. Suporta múltiplos provedores com sistema de créditos pré-pago, session keepers e busca por nome.

🔧 REST JSON 🔑 API Key + JWT 💰 Créditos pré-pago 7 provedores
🔐

Autenticação

A API usa dois métodos de autenticação dependendo do tipo de operação.
ℹ️
API Key — Para renovações e consultas via integração externa. Envie no header X-API-Key.
ℹ️
JWT Token — Para o dashboard. Obtido via login, envie no header Authorization: Bearer TOKEN.
Exemplo com API Key 
curl -X POST "{baseUrl}/renew" \
  -H "Content-Type: application/json" \
  -H "X-API-Key: irapi_live_sua_chave_aqui" \
  -d '{ "provider": "sigma", ... }'
Exemplo com JWT 
curl -X GET "{baseUrl}/auth/profile" \
  -H "Authorization: Bearer eyJhbGciOiJ..."
⚡

Quick Start

Renove um cliente em 3 passos:
✅
1. Faça login e crie uma API Key no dashboard
✅
2. Descubra o sigma_plan_code usando POST /sigma/packages
✅
3. Envie a renovação com POST /renew
Exemplo completo — Renovar por nome 
curl -X POST "{baseUrl}/renew" \
  -H "Content-Type: application/json" \
  -H "X-API-Key: irapi_live_sua_chave" \
  -d '{
    "provider": "sigma",
    "client_name": "João Silva",
    "months": 1,
    "telas": 1,
    "credentials": {
      "username": "meu_user",
      "password": "minha_senha"
    },
    "sigma_domain": "https://painel.exemplo.com",
    "sigma_plan_code": "XYgD9JWr6V"
  }'
🔄

Renovação

Endpoint principal. Cobra créditos, roteia para o microserviço do provedor, e reembolsa automaticamente em caso de falha.
POST /renew API Key 💰 Cobra créditos Renovar cliente ▼

Renova um cliente em qualquer provedor IPTV suportado. Aceita busca por client_name (nome) ou client_id (username). Suporta multi-tela com sufixos.

⚠️
Créditos são debitados antes da renovação e reembolsados automaticamente se houver falha.
ParâmetroTipoDescrição
provider *stringsigma, cloudnation, koffice, uniplay, club, rush, painelfoda
credentials *object{ username, password } — Suas credenciais de revendedor no painel
client_namestringNome do cliente (busca no campo note/nome do painel)
client_idstringUsername do cliente no painel (alternativa ao client_name)
monthsintMeses para renovar (default: 1). No Sigma, o package_id já define a duração.
telasintNúmero de telas/conexões (default: 1)
suffixstringSufixos para multi-tela. Ex: "tela 1,tela 2" → busca "João tela 1", "João tela 2"
sigma_domainstringSigma: URL do painel. Ex: https://painel.sigma.st
sigma_plan_codestringSigma: ID do pacote (use /sigma/packages para descobrir)
koffice_domainstringKoffice: URL do painel
painelfoda_domainstringPainelFoda: URL do painel
painelfoda_package_idstringPainelFoda: ID do pacote
rush_typestringRush: Tipo de plano
user_emailstringE-mail do usuário do gestor que disparou a renovação (para rastreamento). Aparece no histórico.
Exemplo — Sigma por nome 
curl -X POST "{baseUrl}/renew" \
  -H "Content-Type: application/json" \
  -H "X-API-Key: irapi_live_..." \
  -d '{
    "provider": "sigma",
    "client_name": "Fernanda Fernandes",
    "telas": 1,
    "credentials": { "username": "meuUser", "password": "minhaSenha" },
    "sigma_domain": "https://starplay.sigma.st",
    "sigma_plan_code": "ABC123"
  }'
Exemplo — Multi-tela com sufixo 
curl -X POST "{baseUrl}/renew" \
  -H "Content-Type: application/json" \
  -H "X-API-Key: irapi_live_..." \
  -d '{
    "provider": "sigma",
    "client_name": "João",
    "suffix": "tela 1,tela 2,tela 3",
    "telas": 1,
    "credentials": { "username": "meuUser", "password": "minhaSenha" },
    "sigma_domain": "https://starplay.sigma.st",
    "sigma_plan_code": "XYZ789"
  }'
Resposta — Sucesso
{
  "success": true,
  "data": {
    "provider": "sigma",
    "status": "success",
    "client": "Fernanda Fernandes",
    "telas": 1,
    "cost": 0.10,
    "balance": 9.90,
    "response_time_ms": 3240
  }
}

Sigma — Utilitários

Endpoints auxiliares para o provedor Sigma.
POST /sigma/packages JWT / API Key Sem créditos Listar pacotes ▼

Lista todos os pacotes disponíveis em um domínio Sigma. Use para descobrir o sigma_plan_code (package ID) necessário para renovação. Não cobra créditos.

ParâmetroTipoDescrição
credentials *object{ username, password } — Credenciais de revendedor
sigma_domain *stringURL do painel Sigma. Ex: https://painel.sigma.st
Exemplo 
curl -X POST "{baseUrl}/sigma/packages" \
  -H "Content-Type: application/json" \
  -H "X-API-Key: irapi_live_..." \
  -d '{
    "credentials": { "username": "meuUser", "password": "minhaSenha" },
    "sigma_domain": "https://starplay.sigma.st"
  }'
Resposta
{
  "success": true,
  "domain": "https://starplay.sigma.st",
  "servers_count": 2,
  "packages_count": 12,
  "packages": [
    {
      "id": "XYgD9JWr6V",    ← use como sigma_plan_code
      "name": "Mensal 1 Tela",
      "server_name": "Servidor Principal",
      "duration": 1,
      "duration_type": "MONTHS",
      "connections": 1,
      "status": "ACTIVE"
    }
  ]
}

Autenticação & API Keys

POST /auth/login Público Login ▼

Retorna um JWT token para acesso ao dashboard e endpoints protegidos.

ParâmetroTipoDescrição
email *stringE-mail cadastrado
password *stringSenha
Exemplo 
curl -X POST "{baseUrl}/auth/login" \
  -H "Content-Type: application/json" \
  -d '{ "email": "user@email.com", "password": "123456" }'
Resposta
{ "success": true, "token": "eyJhbGciOiJ..." }
GET /auth/profile JWT Perfil + stats ▼

Retorna dados do usuário, saldo de créditos, estatísticas de uso e contagem de API keys.

Exemplo 
curl -H "Authorization: Bearer TOKEN" "{baseUrl}/auth/profile"
GET /auth/api-keys JWT Listar API Keys ▼

Lista todas as API Keys do usuário com data de criação e último uso.

Exemplo 
curl -H "Authorization: Bearer TOKEN" "{baseUrl}/auth/api-keys"
POST /auth/api-keys JWT Criar API Key ▼

Gera uma nova API Key. A chave completa só é retornada uma vez — salve com segurança!

ParâmetroTipoDescrição
name *stringNome da chave (ex: "Gestor Produção")
Exemplo 
curl -X POST "{baseUrl}/auth/api-keys" \
  -H "Authorization: Bearer TOKEN" \
  -H "Content-Type: application/json" \
  -d '{ "name": "Meu Gestor" }'
DEL /auth/api-keys/:id JWT Revogar API Key ▼

Desativa permanentemente uma API Key. A chave não poderá mais ser usada.

Exemplo 
curl -X DELETE "{baseUrl}/auth/api-keys/5" \
  -H "Authorization: Bearer TOKEN"

Créditos

Sistema pré-pago. Cada renovação desconta créditos automaticamente.
GET /credits/balance JWT Consultar saldo ▼
Exemplo 
curl -H "Authorization: Bearer TOKEN" "{baseUrl}/credits/balance"
Resposta
{ "success": true, "balance": 45.50 }
POST /credits/add JWT Adicionar créditos ▼
ParâmetroTipoDescrição
amount *numberValor (mínimo R$ 10.00)
descriptionstringDescrição (opcional)
Exemplo 
curl -X POST "{baseUrl}/credits/add" \
  -H "Authorization: Bearer TOKEN" \
  -H "Content-Type: application/json" \
  -d '{ "amount": 50 }'
GET /credits/transactions JWT Histórico de transações ▼

Retorna histórico de débitos, créditos e reembolsos. Query param: ?limit=30

Exemplo 
curl -H "Authorization: Bearer TOKEN" "{baseUrl}/credits/transactions?limit=50"
📊

Uso & Provedores

GET /usage/summary JWT Resumo do mês ▼

Retorna total de renovações, gastos, e breakdown por provedor no mês atual.

Exemplo 
curl -H "Authorization: Bearer TOKEN" "{baseUrl}/usage/summary"
ℹ️
Também acessível via API Key em GET /usage/api/summary
GET /usage/history JWT Histórico de renovações ▼

Logs detalhados de cada renovação com status, custo, tempo de resposta. Query: ?limit=50&provider=sigma&status=success

Exemplo 
curl -H "Authorization: Bearer TOKEN" "{baseUrl}/usage/history?limit=20&provider=sigma"
ℹ️
Também acessível via API Key em GET /usage/api/history
GET /usage/providers/pricing Público Preços dos provedores ▼

Lista todos os provedores com preços, modo de cobrança (por tela ou por operação) e features.

Exemplo 
curl "{baseUrl}/usage/providers/pricing"
GET /usage/providers/status Público Status dos serviços ▼

Health check de todos os microserviços. Retorna online/offline e tempo de resposta.

Exemplo 
curl "{baseUrl}/usage/providers/status"
👑

Admin

Endpoints exclusivos para administradores. Requerem JWT de um usuário com is_admin = true.
GET /admin/dashboard Admin Dashboard geral ▼

Visão geral: total de usuários, renovações, receita, créditos em circulação.

Exemplo 
curl -H "Authorization: Bearer ADMIN_TOKEN" "{baseUrl}/admin/dashboard"
GET /admin/users Admin Listar usuários ▼

Lista todos os usuários com saldo, total de renovações e status.

Exemplo 
curl -H "Authorization: Bearer ADMIN_TOKEN" "{baseUrl}/admin/users"
POST /admin/users Admin Criar usuário ▼
ParâmetroTipoDescrição
name *stringNome do usuário
email *stringE-mail
password *stringSenha (mín. 6 caracteres)
Exemplo 
curl -X POST "{baseUrl}/admin/users" \
  -H "Authorization: Bearer ADMIN_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{ "name": "João", "email": "joao@email.com", "password": "123456" }'
PUT /admin/users/:id/toggle Admin Ativar/desativar ▼

Alterna entre ativo/inativo. Usuário inativo não pode fazer login nem usar API Keys.

Exemplo 
curl -X PUT "{baseUrl}/admin/users/3/toggle" \
  -H "Authorization: Bearer ADMIN_TOKEN"
POST /admin/users/:id/credits Admin Adicionar créditos ▼
ParâmetroTipoDescrição
amount *numberValor a adicionar
descriptionstringMotivo (ex: "Pagamento PIX")
Exemplo 
curl -X POST "{baseUrl}/admin/users/3/credits" \
  -H "Authorization: Bearer ADMIN_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{ "amount": 50, "description": "PIX recebido" }'
GET /admin/renewals Admin Logs globais ▼

Histórico de todas as renovações de todos os usuários. Query: ?limit=50&user_id=3&provider=sigma

Exemplo 
curl -H "Authorization: Bearer ADMIN_TOKEN" "{baseUrl}/admin/renewals?limit=100"
GET /admin/services Admin Status microserviços ▼

Health check detalhado de todos os microserviços com sessões ativas e features.

Exemplo 
curl -H "Authorization: Bearer ADMIN_TOKEN" "{baseUrl}/admin/services"
ℹ️
Outros endpoints: GET /admin/services/sessions, DELETE /admin/services/sessions, GET /admin/services/:provider/sessions, DELETE /admin/services/:provider/sessions
